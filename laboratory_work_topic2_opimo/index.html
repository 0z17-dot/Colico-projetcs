Design Thinking & User-Centered Design Project

Campus Navigation Solution

Problem Domain

University students struggle to find their way around campus, especially new students and visitors who often get lost or arrive late to classes.

Empathy Phase - User Research

User Persona: Sarah - First Year Student

  • Age: 18
  • Background: New to campus, anxious about getting lost
  • Goals: Find classrooms quickly, avoid being late
  • Pain Points: Complex building layouts, unclear signage, no indoor navigation

Empathy Map

Says: "I always get lost in the engineering building"
Thinks: "Everyone else seems to know where they're going"
Does: Uses Google Maps, asks strangers for directions
Feels: Frustrated, anxious, embarrassed

Problem Statement

How might we help university students navigate campus buildings efficiently so they can arrive at their destinations on time without stress?

Ideation - Possible Solutions

1. AR Campus Navigator App

Uses phone camera to overlay directions on real-world view

2. QR Code Wayfinding System

QR codes at key locations linking to digital maps

3. Smart Kiosk Maps

Interactive touchscreen maps at building entrances

4. Voice-Activated Assistant

"Hey Campus, where is room 301?"

5. Color-Coded Pathways

Physical colored lines on floors leading to different areas

6. Peer Navigator Program

Student volunteers helping newcomers find their way

User Testing Results

Key Findings from User Testing

  • Positive: Users loved the AR feature for complex buildings
  • Improvement: Need clearer visual indicators for floor changes
  • Accessibility: Voice commands helpful for visually impaired users
  • Next Steps: Add offline mode and integration with class schedules

Conclusion & Next Steps

This project demonstrates how Design Thinking and User-Centered Design can work together to solve real campus navigation challenges. The iterative process helped identify user needs and create a solution that addresses actual pain points.

Next Steps:

  • Develop high-fidelity prototype
  • Conduct usability testing with larger user group
  • Implement feedback from testing phase
  • Create technical specifications for development
